# 贡献指南

感谢你考虑为 Merkle Tree 项目贡献代码！

## 如何贡献

### 报告 Bug
1. 在 [Issues](https://github.com/你的账户/merkle-tree003/issues) 页面搜索是否已存在相同问题
2. 若无，创建新 Issue，描述：
   - 问题现象与复现步骤
   - 预期行为
   - 运行环境（Python 版本、操作系统）

### 提交功能建议
- 在 Issues 中标记为 `enhancement`
- 说明功能用途与使用场景

### 提交代码
1. Fork 本仓库到你的账户
2. 克隆到本地：`git clone https://github.com/你的账户/merkle-tree003.git`
3. 创建分支：`git checkout -b feature/your-feature-name`
4. 编写代码并添加必要的测试（若有）
5. 提交：`git commit -m "feat: add your feature description"`
6. 推送：`git push origin feature/your-feature-name`
7. 在 GitHub 上创建 Pull Request

## 代码规范
- 遵循 [PEP 8](https://peps.python.org/pep-0008/) Python 代码风格
- 函数/类添加清晰的 docstring
- 变量命名语义化
- 保持代码简洁可读

## 开发流程
1. 确保本地能运行 `python examples/demo.py`
2. 修改代码后本地测试通过
3. 提交前检查代码风格与导入

## 社区规范
- 尊重他人，友善交流
- Issue 与 PR 描述清晰、有建设性
- 接受 Code Review 反馈并积极改进

## 许可协议
